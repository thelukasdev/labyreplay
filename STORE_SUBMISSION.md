# LabyMod Store Submission Checklist

This repository is prepared for LabyMod Store review using the official LabyMod 4 addon template.

## Required Before Submission

- GitHub Actions build passes on the default branch.
- Release artifact is generated by the official workflow.
- Addon metadata in `build.gradle.kts` is correct:
  - namespace: `labyreplay`
  - display name: `LabyReplay`
  - author: `lukaspellny`
- Source code is public.
- License is present.
- Privacy document is present.

## Review Notes

- The replay core is local-only and does not upload files.
- The addon does not automate clicks, movement, aiming, or gameplay decisions.
- The addon does not bypass server feature restrictions.
- Version-specific game access must stay outside the replay core.
- Mixins may only be added for missing capture hooks and must be documented at the source location.

## Current Implementation Status

- Addon entrypoint and configuration are present.
- Core replay buffer, highlight storage, chat-message detection logic, stats tracking, and share-image rendering are present.
- The next implementation step is wiring the core to LabyMod runtime events:
  - `ChatReceiveEvent` for server chat highlight detection.
  - screenshot/render capture APIs for frame sampling.
  - optional user command or setting action for manual highlight saving.

Do not submit to the Store as feature-complete until the event wiring has been compiled and tested in the LabyMod development client.
